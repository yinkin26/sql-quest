🔗 FOREIGN KEY

🔗 The bridge between tables! FOREIGN KEY creates relationships and maintains data integrity across your database.

📌 What is a FOREIGN KEY?

A FOREIGN KEY is a constraint that creates a link between two tables. It's like a 🌉 bridge that connects related data across different tables. The foreign key in one table points to a primary key in another table, ensuring referential integrity!

🎯 FOREIGN KEY Rules

🔒 Key Rules

  • 🎯 REFERENCES - Must point to a PRIMARY KEY or UNIQUE column
  • 🔍 VALID VALUES - Can only contain values that exist in referenced table
  • 🔄 NULL ALLOWED - Can be NULL (unless NOT NULL specified)
  • 🛡️ REFERENTIAL INTEGRITY - Prevents orphaned records
  • 🔗 CASCADE OPTIONS - Controls what happens when referenced data changes

🧠 Basic Syntax

Method 1: Column-Level Definition

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT REFERENCES customers(customer_id),
    order_date DATE,
    total_amount DECIMAL(10,2)
);

Method 2: Table-Level Definition

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    order_date DATE,
    total_amount DECIMAL(10,2),
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

Method 3: Named Constraint

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    order_date DATE,
    total_amount DECIMAL(10,2),
    CONSTRAINT fk_orders_customer 
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

🔄 CASCADE Options

Control what happens when referenced data is modified:

🗑️ ON DELETE CASCADE

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    order_date DATE,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
    ON DELETE CASCADE  -- 🗑️ Delete orders when customer is deleted
);

🔄 ON UPDATE CASCADE

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    order_date DATE,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
    ON UPDATE CASCADE  -- 🔄 Update orders when customer_id changes
);

🚫 ON DELETE RESTRICT

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,
    order_date DATE,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
    ON DELETE RESTRICT  -- 🚫 Prevent deletion if orders exist
);

🔄 ON DELETE SET NULL

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    customer_id INT,  -- Must allow NULL
    order_date DATE,
    FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
    ON DELETE SET NULL  -- 🔄 Set to NULL when customer deleted
);

📋 CASCADE Options Showcase

🔧 Adding FOREIGN KEY to Existing Table

-- Add foreign key to existing table
ALTER TABLE orders
ADD CONSTRAINT fk_orders_customer
FOREIGN KEY (customer_id) REFERENCES customers(customer_id);

-- Drop foreign key
ALTER TABLE orders
DROP FOREIGN KEY fk_orders_customer;

🌍 Real-World FOREIGN KEY Examples

🛒 E-commerce System

-- Products table
CREATE TABLE products (
    product_id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    price DECIMAL(10,2)
);

-- Order items with foreign keys
CREATE TABLE order_items (
    item_id INT PRIMARY KEY AUTO_INCREMENT,
    order_id INT,
    product_id INT,
    quantity INT,
    price DECIMAL(10,2),
    FOREIGN KEY (order_id) REFERENCES orders(order_id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES products(product_id) ON DELETE RESTRICT
);

📚 Blog System

-- Users table
CREATE TABLE users (
    user_id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE,
    email VARCHAR(100)
);

-- Posts with author reference
CREATE TABLE posts (
    post_id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(200),
    content TEXT,
    author_id INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (author_id) REFERENCES users(user_id) ON DELETE SET NULL
);

-- Comments with post and user references
CREATE TABLE comments (
    comment_id INT PRIMARY KEY AUTO_INCREMENT,
    post_id INT,
    user_id INT,
    comment_text TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (post_id) REFERENCES posts(post_id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(user_id) ON DELETE SET NULL
);

🏫 School Management

-- Students and courses with enrollment
CREATE TABLE students (
    student_id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    email VARCHAR(100)
);

CREATE TABLE courses (
    course_id INT PRIMARY KEY AUTO_INCREMENT,
    course_name VARCHAR(100),
    credits INT
);

CREATE TABLE enrollments (
    enrollment_id INT PRIMARY KEY AUTO_INCREMENT,
    student_id INT,
    course_id INT,
    enrollment_date DATE,
    grade CHAR(2),
    FOREIGN KEY (student_id) REFERENCES students(student_id) ON DELETE CASCADE,
    FOREIGN KEY (course_id) REFERENCES courses(course_id) ON DELETE CASCADE
);

⚠️ FOREIGN KEY Best Practices

✅ Do's

  • 📝 Name constraints - use descriptive names like 'fk_orders_customer'
  • 🎯 Choose appropriate CASCADE - think about business logic
  • 📊 Index foreign keys - improves JOIN performance
  • 🔍 Validate data first - ensure referenced records exist
  • 📋 Document relationships - maintain clear ER diagrams

❌ Don'ts

  • 🚫 Don't create circular references - avoid table A → B → A
  • ⚡ Don't over-cascade - be careful with CASCADE DELETE
  • 🔄 Don't ignore orphaned data - clean up before adding constraints
  • 📏 Don't reference non-indexed columns - impacts performance
  • 🔗 Don't forget to handle NULLs - decide if foreign keys can be NULL

💡 Helpful Tips

  • 🔍 Use SHOW CREATE TABLE table_name; to see foreign key definitions
  • 📊 Performance - foreign keys automatically create indexes in some databases
  • 🔗 Relationships - foreign keys enable JOINs between tables
  • 🛡️ Data integrity - prevents invalid references
  • 🔄 CASCADE carefully - test cascade behavior thoroughly
  • 📋 Naming convention - use 'fk_childtable_parenttable' pattern

🧠 Summary

  • 🔗 FOREIGN KEY creates relationships between tables
  • 🎯 References primary key or unique column in another table
  • 🛡️ Referential integrity - prevents orphaned records
  • 🔄 CASCADE options control behavior when referenced data changes
  • ⚡ Performance impact - enables efficient JOINs
  • 📊 Data modeling - essential for normalized database design

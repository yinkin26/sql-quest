🚀 SQL Indexes

🚀 The speed boosters! Indexes are like a book's index - they help find data quickly without scanning every page.

📌 What are Indexes?

An INDEX is a data structure that improves the speed of data retrieval operations on a table. Think of it like a 📚 library catalog that helps you find books quickly without searching every shelf!

🎯 Index Features

🔍 Key Features

  • ⚡ FASTER QUERIES - Dramatically speeds up SELECT operations
  • 🎯 EFFICIENT SORTING - Speeds up ORDER BY clauses
  • 🔍 QUICK SEARCHES - Accelerates WHERE clause filtering
  • 🔗 FASTER JOINS - Improves performance of table joins
  • 📊 UNIQUE ENFORCEMENT - Can enforce uniqueness constraints

🧠 Index Types

1. Primary Index (Clustered)

-- Automatically created with PRIMARY KEY
CREATE TABLE users (
    user_id INT PRIMARY KEY AUTO_INCREMENT,  -- 🔑 Clustered index
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL
);

2. Secondary Index (Non-Clustered)

-- Create index on frequently searched columns
CREATE INDEX idx_username ON users(username);
CREATE INDEX idx_email ON users(email);

3. Unique Index

-- Ensures uniqueness and provides fast lookups
CREATE UNIQUE INDEX idx_unique_email ON users(email);

-- Alternative: Add unique constraint (creates unique index automatically)
ALTER TABLE users ADD CONSTRAINT uk_username UNIQUE (username);

4. Composite Index (Multi-Column)

-- Index on multiple columns (order matters!)
CREATE INDEX idx_name_age ON users(last_name, first_name, age);

-- Good for queries like:
-- WHERE last_name = 'Smith'
-- WHERE last_name = 'Smith' AND first_name = 'John'
-- WHERE last_name = 'Smith' AND first_name = 'John' AND age = 25

5. Partial Index

-- Index only specific rows (with WHERE condition)
CREATE INDEX idx_active_users ON users(username) 
WHERE status = 'active';

-- Index only non-NULL values
CREATE INDEX idx_phone_not_null ON users(phone) 
WHERE phone IS NOT NULL;

🌍 Real-World Index Examples

👥 User Management System

CREATE TABLE users (
    user_id INT PRIMARY KEY AUTO_INCREMENT,     -- 🔑 Clustered index (automatic)
    username VARCHAR(50) UNIQUE NOT NULL,       -- 🎯 Unique index (automatic)
    email VARCHAR(100) UNIQUE NOT NULL,         -- 🎯 Unique index (automatic)
    first_name VARCHAR(50) NOT NULL,
    last_name VARCHAR(50) NOT NULL,
    date_of_birth DATE,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login TIMESTAMP
);

-- Additional indexes for performance
CREATE INDEX idx_users_name ON users(last_name, first_name);  -- 👤 Name searches
CREATE INDEX idx_users_status ON users(status);                    -- 📊 Status filtering
CREATE INDEX idx_users_created ON users(created_at);                -- 📅 Date range queries
CREATE INDEX idx_users_last_login ON users(last_login);              -- 🕐 Login tracking
CREATE INDEX idx_users_birth_year ON users(YEAR(date_of_birth));     -- 🎂 Age-based queries

🛒 E-commerce Product System

CREATE TABLE products (
    product_id INT PRIMARY KEY AUTO_INCREMENT,  -- 🔑 Clustered index
    sku VARCHAR(50) UNIQUE NOT NULL,            -- 🎯 Unique index
    name VARCHAR(200) NOT NULL,
    description TEXT,
    price DECIMAL(10,2) NOT NULL,
    category_id INT NOT NULL,
    brand_id INT NOT NULL,
    stock_quantity INT DEFAULT 0,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- Performance indexes
CREATE INDEX idx_products_category ON products(category_id);         -- 📂 Category filtering
CREATE INDEX idx_products_brand ON products(brand_id);               -- 🏢 Brand filtering
CREATE INDEX idx_products_price ON products(price);                  -- 💰 Price sorting/filtering
CREATE INDEX idx_products_status ON products(status);                -- 📊 Status filtering
CREATE INDEX idx_products_name ON products(name);                    -- 🔍 Name searching
CREATE INDEX idx_products_stock ON products(stock_quantity);         -- 📦 Inventory queries

-- Composite indexes for common query patterns
CREATE INDEX idx_products_cat_price ON products(category_id, price); -- 📂💰 Category + price
CREATE INDEX idx_products_brand_status ON products(brand_id, status); -- 🏢📊 Brand + status
CREATE INDEX idx_products_price_stock ON products(price, stock_quantity); -- 💰📦 Price + stock

⚠️ Index Best Practices

✅ Do's

  • 🎯 Index WHERE columns - columns frequently used in WHERE clauses
  • 🔗 Index JOIN columns - foreign keys and join conditions
  • 📊 Index ORDER BY columns - columns used for sorting
  • 🎪 Composite indexes - for multi-column queries (order matters!)
  • 📈 Monitor performance - use EXPLAIN to verify index usage
  • 🔍 Selective columns - index columns with high cardinality

❌ Don'ts

  • 🚫 Don't over-index - too many indexes slow down INSERT/UPDATE/DELETE
  • 📝 Don't index small tables - overhead may outweigh benefits
  • 🔄 Don't index frequently changing columns - high maintenance cost
  • 📊 Don't index low-cardinality columns - like boolean or status with few values
  • 🎯 Don't ignore composite index order - leftmost prefix rule applies
  • 💾 Don't forget storage cost - indexes consume disk space

💡 Helpful Tips

  • 🔍 Use SHOW INDEX FROM table_name; to see all indexes
  • ⚡ Composite index order - most selective column first
  • 🛡️ Leftmost prefix rule - composite indexes work left-to-right
  • 🎯 Covering indexes - include all columns needed by query
  • 🔄 Index maintenance - MySQL automatically maintains indexes
  • 📊 Monitor usage - remove unused indexes to improve write performance
  • 🚨 Index size - keep an eye on index storage overhead

🧠 Summary

  • 🚀 Indexes dramatically improve query performance
  • 🎯 Strategic placement - index columns used in WHERE, JOIN, ORDER BY
  • ⚡ Multiple types - single, composite, unique, partial indexes
  • 🛡️ Automatic maintenance - MySQL keeps indexes up-to-date
  • 📊 Trade-offs - faster reads vs slower writes and more storage
  • 🔧 Optimization tool - use EXPLAIN to verify index usage

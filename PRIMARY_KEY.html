🔑 PRIMARY KEY

🔑 The master key to your data! PRIMARY KEY is the unique identifier that makes each row special.

📌 What is a PRIMARY KEY?

A PRIMARY KEY is a constraint that uniquely identifies each row in a table. It's like a 🆔 social security number for your data - no two rows can have the same primary key value. Think of it as the 🏠 address of each record in your database neighborhood!

🎯 PRIMARY KEY Rules

🔒 Key Rules

  • 🔑 UNIQUE - No duplicate values allowed
  • 🚫 NOT NULL - Cannot be empty or NULL
  • 1️⃣ ONE PER TABLE - Only one primary key per table
  • 🔒 IMMUTABLE - Should not change once set
  • ⚡ AUTO INDEX - Automatically creates an index for fast lookups

🧠 Basic Syntax

Method 1: Column-Level Definition

CREATE TABLE users (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(100)
);

Method 2: Table-Level Definition

CREATE TABLE users (
    id INT,
    name VARCHAR(100),
    email VARCHAR(100),
    PRIMARY KEY (id)
);

Method 3: AUTO_INCREMENT Primary Key

CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    email VARCHAR(100)
);

🔗 Composite PRIMARY KEY

Sometimes you need multiple columns to uniquely identify a row:

CREATE TABLE order_items (
    order_id INT,
    product_id INT,
    quantity INT,
    price DECIMAL(10,2),
    PRIMARY KEY (order_id, product_id)
);

🔧 Adding PRIMARY KEY to Existing Table

-- Add primary key to existing table
ALTER TABLE existing_table
ADD CONSTRAINT pk_existing_table PRIMARY KEY (id);

-- Drop primary key (if needed)

🌍 Real-World PRIMARY KEY Examples

👥 User Management

CREATE TABLE users (
    user_id INT PRIMARY KEY AUTO_INCREMENT,  -- 🔑 Unique user identifier
    username VARCHAR(50) UNIQUE NOT NULL,        -- ⭐ Unique username
    email VARCHAR(100) UNIQUE NOT NULL,          -- 📧 Unique email
    password_hash VARCHAR(255) NOT NULL,         -- 🔒 Encrypted password
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

🛒 E-commerce Orders

CREATE TABLE orders (
    order_id INT PRIMARY KEY AUTO_INCREMENT,  -- 🔑 Unique order number
    customer_id INT NOT NULL,                    -- 👤 Link to customer
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    total_amount DECIMAL(10,2) NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',
    FOREIGN KEY (customer_id) REFERENCES users(user_id)
);

📚 Library System

CREATE TABLE books (
    isbn VARCHAR(13) PRIMARY KEY,            -- 📖 ISBN as natural primary key
    title VARCHAR(200) NOT NULL,
    author VARCHAR(100) NOT NULL,
    publication_year INT,
    available_copies INT DEFAULT 0
);

🆔 Natural vs Surrogate Keys

🌿 Natural Primary Key

Uses existing meaningful data:

-- Using Social Security Number
CREATE TABLE employees (
    ssn VARCHAR(11) PRIMARY KEY,  -- 🆔 Natural identifier
    first_name VARCHAR(50),
    last_name VARCHAR(50)
);

🤖 Surrogate Primary Key

Uses artificial auto-generated values:

-- Using auto-increment ID
CREATE TABLE employees (
    employee_id INT PRIMARY KEY AUTO_INCREMENT,  -- 🤖 Artificial identifier
    ssn VARCHAR(11) UNIQUE,
    first_name VARCHAR(50),
    last_name VARCHAR(50)
);

⚠️ PRIMARY KEY Best Practices

✅ Do's

  • 🤖 Use surrogate keys (AUTO_INCREMENT) for most tables
  • 📏 Keep it simple - prefer single column primary keys
  • 🔢 Use integers when possible for better performance
  • 📝 Name consistently - use table_name + '_id' pattern
  • 🎯 Always define a primary key for every table

❌ Don'ts

  • 🚫 Don't use business data as primary keys (emails, names)
  • 🔄 Don't change primary key values once set
  • 📏 Don't make them too long - impacts performance
  • 🔤 Avoid string primary keys when possible
  • 🚫 Never leave tables without primary keys

💡 Helpful Tips

  • 🔍 Use DESCRIBE table_name; to see which column is the primary key
  • ⚡ Performance boost - primary keys automatically create clustered indexes
  • 🔗 Foreign key target - other tables reference primary keys
  • 🔢 AUTO_INCREMENT starts at 1 and increments automatically
  • 🎯 Replication friendly - surrogate keys work better in distributed systems
  • 📊 Query optimization - database engines optimize queries using primary keys

🧠 Summary

  • 🔑 PRIMARY KEY uniquely identifies each row in a table
  • 🚫 Must be UNIQUE and NOT NULL
  • 1️⃣ One per table - each table can have only one primary key
  • 🤖 AUTO_INCREMENT automatically generates unique values
  • ⚡ Performance benefits - creates automatic indexes
  • 🔗 Relationship foundation - used by foreign keys to link tables

üìå What are SQL Data Types?

SQL data types define the type of data that can be stored in each column of a table. They help the database understand how to store, search, and manipulate the data effectively. Choosing the correct data type for each column is essential for efficiency, accuracy, and data integrity.

There are many types of data in SQL, but we‚Äôll cover the most commonly used ones.

üß† Common SQL Data Types

1. Numeric Data Types

SQL provides several data types to store numbers, both integers (whole numbers) and decimals, like INT, DECIMAL, and FLOAT.

Data Type Description Example
INT or INTEGER Used to store whole numbers without any decimals. 42, -5
DECIMAL or NUMERIC Used to store precise decimal numbers. Useful for currency and other calculations that require precision. 15.99, -500.5
FLOAT Stores approximate floating-point numbers with decimal points. Precision may vary based on system. 3.1415, 2.71
DOUBLE Stores double-precision floating-point numbers. Used for very large or very precise numbers. 3.14159265358979

Tip: Use INT for whole numbers and DECIMAL/NUMERIC for precise decimal values like prices or measurements.

2. Textual Data Types

Text-based data types are used to store strings of characters like names, descriptions, or comments, such as VARCHAR and CHAR.

Data Type Description Example
VARCHAR(n) Stores variable-length strings. The n specifies the maximum length of the string. 'John Doe', 'apple'
CHAR(n) Stores fixed-length strings. The n specifies the length. If the string is shorter than n, it will be padded with spaces. 'John' (with length 5)
TEXT Stores long text without a specified length limit. Used for large texts like descriptions or comments. 'This is a long paragraph...'

Tip: Use VARCHAR for varying string lengths and CHAR when the length is fixed.

3. Date and Time Data Types

Use DATE for dates, TIME, and DATETIME or TIMESTAMP.

Data Type Description Example
DATE Stores a date in the format YYYY-MM-DD. '2025-04-02'
TIME Stores a time in the format HH:MM:SS. '14:30:00'
DATETIME Stores both date and time in the format YYYY-MM-DD HH:MM:SS. '2025-04-02 14:30:00'
TIMESTAMP Similar to DATETIME but often used for automatically recording when a record was created or updated. '2025-04-02 14:30:00'

Tip: Use DATE for storing just dates (e.g., birthdates), TIME for times (e.g., working hours), and DATETIME or TIMESTAMP for both.

4. Boolean Data Type

The BOOLEAN data type is used to store true/false values.

Data Type Description Example
BOOLEAN Stores a boolean value, typically TRUE or FALSE. TRUE, FALSE

Tip: In SQL, TRUE and FALSE are often stored as 1 (TRUE) and 0 (FALSE).

5. Binary Data Types

Binary data types are used to store binary data (like images or files), such as BLOB and BINARY.

Data Type Description Example
BLOB Stores binary large objects, like images, audio, or other files. image.jpg, audio.mp3
BINARY(n) Stores fixed-length binary data (n specifies length). 01010101 (8-bit binary)

Tip: Use BLOB for large binary files and BINARY for fixed-size binary data like flags.

üîÑ Working with Data Types: Practical Examples

Example 1: Creating a Table

CREATE TABLE employees (
  id INT PRIMARY KEY,
  name VARCHAR(100),
  salary DECIMAL(10, 2),
  hire_date DATE,
  is_active BOOLEAN
);

üìö Recap

  • SQL data types define what kind of data can be stored in a table.
  • Numeric types: Store numbers (integers or decimals).
  • Text types: Store strings of text.
  • Date and time types: Store dates and timestamps.
  • Boolean: Store true or false values.
  • Binary: Store files like images or audio.

Make sure to choose the correct data type for your table columns based on the type of data you need to store!

üìå What is LIMIT / OFFSET?

The LIMIT and OFFSET clauses are used to control the number of rows returned in a SQL query, allowing for efficient pagination of large datasets.

üß† Basic Syntax

SELECT column1, column2 FROM table_name
LIMIT number_of_rows OFFSET start_position;

üí° Example

SELECT name, city FROM customers
LIMIT 5 OFFSET 10;

üåç Real-World Use Case

SQL LIMIT and OFFSET are commonly used in applications to implement pagination. For example, an e-commerce website might use these clauses to display products 10 at a time. Here's how you can display 10 products starting from the 11th:

SELECT product_name, price FROM products
LIMIT 10 OFFSET 10;

üí° Helpful Tips

  • LIMIT specifies the maximum number of rows to return.
  • OFFSET skips a number of rows before returning results (use with LIMIT).
  • ‚úÖ MySQL Tip: LIMIT offset, count is also valid. Example:
    SELECT * FROM products LIMIT 2, 3; ‚Üí Skips 2 rows, returns 3
  • ‚ö†Ô∏è Note: OFFSET without LIMIT may not be supported in all databases (e.g., MySQL).

üß† Summary

  • LIMIT restricts the number of rows returned by a query.
  • OFFSET allows you to skip a specified number of rows before returning the remaining rows.
  • Use both clauses to paginate query results efficiently.

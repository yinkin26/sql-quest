🚫 NOT NULL Constraint

🚫 The data guardian! NOT NULL constraint ensures that critical columns always contain meaningful data.

📌 What is a NOT NULL Constraint?

A NOT NULL constraint ensures that a column cannot have an empty (NULL) value. It's like having a 🛡️ security guard that prevents incomplete data from entering your database!

🎯 NOT NULL Constraint Features

🔍 Key Features

  • 🚫 NO EMPTY VALUES - Prevents NULL values in columns
  • 🛡️ DATA INTEGRITY - Ensures critical data is always present
  • ⚡ FAST VALIDATION - Simple and efficient constraint checking
  • 🎯 BUSINESS RULES - Enforces required field logic
  • 📊 QUERY OPTIMIZATION - Helps database optimize queries

🧠 Basic Syntax

Method 1: Column Definition with NOT NULL

CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL,
    password VARCHAR(255) NOT NULL,
    phone VARCHAR(20),  -- This can be NULL
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP NOT NULL
);

Method 2: Adding NOT NULL to Existing Column

-- Add NOT NULL to existing column (must have no NULL values first)
ALTER TABLE users
MODIFY COLUMN phone VARCHAR(20) NOT NULL;

-- Remove NOT NULL from column
ALTER TABLE users
MODIFY COLUMN phone VARCHAR(20) NULL;

Valid and Invalid Data Examples

  • first_name: NULL or empty string
  • last_name: NULL or empty string
  • email: NULL or empty string
  • created_at: NULL (but has DEFAULT)

🔍 NOT NULL vs NULL Behavior

✅ Valid INSERT Operations

-- All required fields provided
INSERT INTO customers (first_name, last_name, email, phone, address)
VALUES ('Alice', 'Johnson', 'alice@email.com', '+1-555-0789', '456 Oak Ave');

-- Optional fields omitted (phone, address can be NULL)
INSERT INTO customers (first_name, last_name, email)
VALUES ('Charlie', 'Brown', 'charlie@email.com');

-- Explicit NULL for optional fields
INSERT INTO customers (first_name, last_name, email, phone, address)
VALUES ('David', 'Miller', 'david@email.com', NULL, NULL);

❌ Invalid INSERT Operations

-- Missing required field (first_name)
INSERT INTO customers (last_name, email)
VALUES ('Smith', 'missing@email.com');  -- ERROR!

-- Explicit NULL for NOT NULL field
INSERT INTO customers (first_name, last_name, email)
VALUES (NULL, 'Johnson', 'null@email.com');  -- ERROR!

-- Empty string might be allowed (depends on SQL mode)
INSERT INTO customers (first_name, last_name, email)
VALUES ('', 'Empty', 'empty@email.com');  -- May work, but not recommended

🔧 Managing NOT NULL Constraints

Adding NOT NULL to Existing Column

-- Step 1: Update existing NULL values (if any)
UPDATE users 
SET phone = 'Unknown' 
WHERE phone IS NULL;

-- Step 2: Add NOT NULL constraint
ALTER TABLE users
MODIFY COLUMN phone VARCHAR(20) NOT NULL;

Removing NOT NULL Constraint

-- Allow NULL values in previously NOT NULL column
ALTER TABLE users
MODIFY COLUMN phone VARCHAR(20) NULL;

Checking for NULL Values

-- Find rows with NULL values
SELECT * FROM users WHERE phone IS NULL;

-- Count NULL values
SELECT COUNT(*) FROM users WHERE phone IS NULL;

-- Find columns that can accept NULL
SELECT COLUMN_NAME, IS_NULLABLE 
FROM INFORMATION_SCHEMA.COLUMNS 
WHERE TABLE_NAME = 'users' AND TABLE_SCHEMA = 'your_database';

⚠️ NOT NULL Constraint Best Practices

✅ Do's

  • 🎯 Critical fields - use NOT NULL for essential business data
  • 🔑 Identifiers - usernames, emails, IDs should be NOT NULL
  • 📊 Status fields - use NOT NULL with DEFAULT values
  • 📅 Timestamps - audit fields should be NOT NULL
  • 💰 Financial data - amounts, prices should be NOT NULL
  • 🛡️ Security fields - passwords, tokens should be NOT NULL

❌ Don'ts

  • 🚫 Don't overuse - not every field needs to be NOT NULL
  • 📞 Don't force optional data - phone numbers, addresses can be optional
  • 🎨 Don't require cosmetic fields - profile pictures, descriptions
  • 🔄 Don't add NOT NULL without planning - consider existing data
  • 📝 Don't use empty strings - prefer NULL for truly missing data

🔍 NULL vs Empty String vs Zero

Understanding the Differences

  • 🚫 NULL - No value, unknown, not applicable
  • 📝 Empty String ('') - Known value that happens to be empty
  • 🔢 Zero (0) - Actual numeric value of zero
  • 🎯 NOT NULL - Prevents NULL, but allows empty strings and zero
-- These are all different:
name = NULL        -- Unknown name
name = ''          -- Known to be empty
age = NULL         -- Unknown age
age = 0            -- Actually zero years old
balance = NULL     -- Unknown balance
balance = 0.00     -- Actually zero dollars
